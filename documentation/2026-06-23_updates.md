# Release Log: June 23, 2026

## Implementations & Changes Completed Today

Today we designed, implemented, and fully integrated several key modules and command interfaces into A.N.A.N.D:

### 1. Multi-Agent Orchestration & Debugging Loop
- **Debugger Agent**: Spawns automatically when one or more files are written/modified by a Coding Agent.
  - Verification: Reads code changes, executes checks/tests, and validates correctness.
  - Self-Healing: Directly fixes small bugs (like typos or syntax syntax) using `<write_file>` tags.
  - Escalation: Writes a detailed error report and hands big issues over to the Commander Agent if the error requires design revisions.
  - Dynamic Models: Configurable by Commander via `debugger_model="model_name"` inside the `<spawn_agent>` tag, or defaulting to user-specified selection.
- **Dynamic Colored Output Streams**:
  - Commander Agent: Green/White text.
  - Coding Agents: Magenta text.
  - Debugger Agent: Yellow text (`chalk.yellow`).

### 2. Command Toggles: Algo vs. Normal Modes
- **Orchestrated Algorithm Mode (`/algo`)**: Runs full multi-agent loop with Commander planning and subagent spawning. The status bar displays `[Mode: ALGO]`.
- **Normal Chatbot Mode (`/normal`)**: Switches to a single-agent experience where users chat directly with A.N.A.N.D. Spawning capability is disabled, but direct capability tools (`<run_command>`, `<read_file>`, `<write_file>`) remain fully functional. The status bar displays `[Mode: NORMAL]`.

### 3. Smart & Searchable Pickers
- **Search Filtering**: Selection menus (`/models`, `/coding-models`, `/debugger`, `/provider`) now contain a real-time `Search > ` input bar. Filtering is case-insensitive, paginated, and updates live as keyboard characters are typed.
- **Defaults Memory**: Automatically remembers and highlights the active provider, active model, or active debugger model on open, removing the need to search or scroll from the top every time.

### 4. Global Installation Access
- Prefixed `harness.js` with `#!/usr/bin/env node` shebang.
- Configured dynamic ESM paths resolving absolute main path relative to the module file.
- Added `"bin"` entry mapping `anand` to `./harness.js` inside `NodeJS/package.json`.
- Ran `npm link` to registers the CLI globally, enabling launcher execution by writing `anand` from any directory.
